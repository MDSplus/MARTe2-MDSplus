Detailed Description

GAM which wraps Python code.

This GAM wraps Python code so that it can be run as a MARTe2 GAM.

By default the GAM checks that the output tuple returned by the Python code is coherent with the layout of the output signals as declared in the configuration file at every cycle. This check can be disabled to improve performances by setting RealtimeOutputCheck parameter to 0. If no RealtimeOutputCheck parameter is specified, the GAM assumes it to be 1.

+MyGAM = {
    Class               = PyGAM
    FileName            = "pythonModuleName"
    RealtimeOutputCheck = 1                   // Set to 0 to disable checks on the output tuple.
    Parameters = {
        param1 = 1
        param2 = {1, 2}
        param3 = { {1, 2}, {3, 4} }
    }
    InputSignals = {
        ...
    {
    OutputSignals = {
        ...
    {
}

... some additional info ...

Precondition
1. Number of inputs and outputs of the configuration file for this GAM must match exactely the number of (respectively) arguments and return values of the Python code the GAM will run.
2. Values of parameters declared in the model must be set in the configuration file. Not setting the value for all paramters returns an error.

bool Setup ( )
virtual
  1. Coherency checks between data provided in MARTe2 configuration file and what is written in the Python code.
  2. Retrieved informations are used to build PyObject tuples that will contain actual input and output data. The tuple will then be used to pass arguments to the execute() function and to store its outputs.
  3. Error handling.
